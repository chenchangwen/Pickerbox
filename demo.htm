特别注意:

1.demo.htm已经调用了pickerbox(),同时iframe引用了pickerbox.htm.pickerbox.htm也调用了pickerbox().这种情况必须指定pickerbox参数的iframeid,如该例子iframeid:'iniframe'.否则将会匹配值出错.

2.点击图标时会弹出数据窗口,数据窗口是ColorBox v1.3.20.1. 当前页面是demo.htm,即调用pickerbox的页面引入ColorBox v1.3.20.1即可,否则弹出的数据窗口会在最后引入的页面调用.

属于调用页面

pickerbox对象参数

属性

名称 类型 描述 默认值
title（标题） string 显示在pickerbox顶部的标题文本. 空字符串
height（高度） number pickerbox的高度 100%
width（宽度） number pickerbox的宽度 空
text（文本） string pickerbox的已选文本 空字符串
value（值） string pickerbox的已选值 空字符串
iframeid（iframe的id） string 如果是被iframe引用的,即父窗口是iframe,必须设定 空字符串
ismultiple（多选） boolean 弹出数据窗口时的数据是否可以多选 true
url（地址） string 弹出数据窗口的地址 空字符串
pickergird object

在数据窗口点击每条记录时,会根据fieldvalue作为值,fieldtext作为文本.

如有json {"RoleId":"1001","RoleName":"角色1","RoleId2":"10012","RoleName2":"角色2"}

pickergird:{ fieldtext:'RoleName', fieldvalue:'RoleId' }

那么点击时,将保存文本:"角色1",值:"1001".

 空字符串
onselected function(param)

当在数据窗口点击确认时,会在该函数返回一个'param'对象.

'param'参数包含以下属性：

id:字符串,当前对象的id.

value:字符串,当前对象已选的值.

text:字符串,当前对象已选的文本.

 空字符串
showtype string

pickerbox显示的样式

'string'参数为以下字符串：

'select' 显示为select样式

'input' 显示为input样式

select

方法

名称 参数 描述
$.pickerbox.getCountById string(pickerbox的id) 返回已选数量.
$.pickerbox.getTextById string(pickerbox的id) 返回已选文本.
$.pickerbox.getValueById string(pickerbox的id) 返回已选值.

一个用以从远程站点请求数据的超链接地址.

可参考datagrid_data1.json和datagrid_data2.json

如:{"total":10,"rows":[{"DeptId":"0001","DeptName":"行政部"}]}

total:总记录数.

rows:每行数据对应的json对象.

 空字符串

分页参数

名称 描述
page 第几页
rows 行数

分页的数量超过当页设定的数量时,分页按钮即启用.当点[下一页](等翻页按钮)时.会发送参数到后台(地址是当前加载的url).

那么数据表格请求地址如:http://localhost/pickerbox/handler.aspx?page=1&row=10

所以,后台应根据这参数做出相应处理.即返回正确的JSON数据.

picker对象

方法

名称 参数 描述
build object 根据picker对象参数,创建数据窗口内容.
reload where

where 应该是一个正确的json字符串.

数据表格会根据当前url并附加上where重新加载请求.

如:

var where = { 'DeptName': '行政部' };

var result = baidu.json.stringify(where);

picker.reload(result);

例如这里使用了baidu.json.stringify进行json格式化.

那么数据表格请求地址将是如:

http://localhost/pickerbox/handler.aspx?where=%7B%22DeptName%22%3A%22%E8%A1%8C%E6%94%BF%E9%83%A8%22%7D&page=1&rows=10

会对请求内容进行编码.

那么后台就可以获取参数where.

关于pickerbox

pickerbox简称采摘盒子,基于Jquery的插件,引用了JQueryzTree-v3.5.12,easyui1.3.2,及ColorBox v1.3.20.1.只为更方便的取得简单的或多条件的数据.

-支持树,表格呈现数据.

-支持跨iframe获取数据.

-灵活支持记录数据.

-支持重要事件回调.
